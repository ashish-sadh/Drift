# Drift Daily Briefing — 2026-04-18

## Headline
AI chat quality hardened with automated evaluation tests and voice input improvements — the pipeline is now self-testing across 400+ scenarios.

## Product Snapshot
Drift is an AI-first health tracker — users type what they ate or did, AI handles logging, tracking, and insights. On-device, no cloud. Currently in closed beta on TestFlight.

## What Shipped This Period
- **Voice logging is cleaner** — filler words (um, uh, like, so, you know) are now stripped before AI processing, so "um I had like 2 eggs" logs cleanly
- **AI pipeline is now self-testing** — new Karpathy-style auto-research optimizer finds the best prompt configuration automatically and applies it when accuracy improves
- **Food database expanded** — added Burger King, Subway, Domino's items, Indian snacks (Gathiya, Surti Locho, Sev Mamra), premium yogurts, and supplement powders (+20 items, now 2,207 total)
- **Chat bugs fixed** — "a cup of dal" now parses correctly; "log lunch" no longer mis-logs a previously mentioned food
- **Dashboard alerts cleaned up** — duplicate alerts removed; protein and workout alerts suppressed for users who haven't started logging yet

## What's Working Well
- AI eval harness is the strongest it's ever been — 400+ deterministic scenarios + 130-case LLM gold set, all at 100% pass rate
- Voice input end-to-end is stable — stuck spinner, word loss, and filler noise all fixed in back-to-back builds
- Automated prompt optimization means AI quality can improve without manual tuning sessions

## Risks & Blockers
- Multi-turn food logging (adding items across multiple chat turns) still needs a reliability test pass — scheduled for this sprint but not yet verified against the live LLM
- Supplement intent disambiguation ("did I take creatine" vs "log creatine") is covered by deterministic tests but LLM routing is only partially verified
- Food database at 2,207 is 0.016% of MyFitnessPal's scale — still a long road to parity on food coverage

## Focus for Next Period
1. **Verify multi-turn reliability with live LLM** — the 3-turn breakfast continuation test is written; run it against the eval models and fix any failures via prompt tuning
2. **Supplement status routing** — confirm "did I take creatine?" consistently routes to status (not intake logging) across the LLM eval suite
3. **Prompt token audit** — current intent classifier prompt is verbose; compress redundant examples to cut token count ≥15% with no accuracy drop
