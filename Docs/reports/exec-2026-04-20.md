# Drift Daily Briefing — 2026-04-20

## Headline
Extractor reliability & cross-domain chat sprint landed — intent confidence v2, cross-domain pronoun resolution, auto-retry on empty extraction, plus DomainExtractor 50-case gold set to lock the gains.

## Product Snapshot
Drift is an AI-first health tracker — users type what they ate or did, AI handles logging, tracking, and insights. On-device, no cloud. Currently in closed beta on TestFlight.

## What Shipped This Period
- **Cross-domain pronoun resolution** — "what was that?" after a weight log now resolves across food + weight context, not just within-domain (MVP, #241)
- **Intent confidence v2** — classifier now uses positive-signal gates before entering clarifier flow; fewer unnecessary "what do you mean?" interruptions on clear queries (#242)
- **Auto-retry on empty extraction** — when the first extraction pass returns empty/incomplete JSON, the pipeline automatically re-prompts with a literal hint instead of falling back to generic chat
- **Food DB +60** — boba/bubble tea, poke bowls, açaí bowls, specialty coffee, sports nutrition, Vietnamese expanded (now 2,511 total)
- **Eval coverage expanded** — DomainExtractor 50-case gold set (isolated food/weight/exercise extraction), +25 hard food-logging cases (edge units, implicit qty, composed foods), +15 multi-turn regression cases (ordinal refs, mid-recipe topic switch, partial undo)
- **Latency benchmarks hardened** — ChatLatencyBenchmark now covers confirmation-card and clarify-round-trip scenarios, so the regression guard catches slowdowns beyond just first-token time
- **Gemma 4 download hardened** — retry, size validation, pinned revision to prevent partial-download breakage (#238)

## What's Working Well
- AI sprint discipline is compounding — each cycle lands 2-3 real pipeline improvements, locked behind gold-set tests that prevent regression
- Per-stage isolation testing (pipeline Stage 1/2/3/4/5 as separate evals) is catching real bugs that end-to-end tests miss
- Test suite crossed 1,677 — every new AI feature now ships with regression-blocking coverage

## Risks & Blockers
- Food DB at 2,511 but still short of the long tail — need USDA fallback wired to chat tier, not just browse, to cover queries like "what's in Starbucks cold brew"
- Multi-turn reliability still the weakest link: ordinal references ("log the second one") and mid-turn topic switches still trip the classifier on ~15% of hard cases
- No production user telemetry — all eval signals come from synthetic gold sets; we could be improving on the wrong axis

## Focus for Next Period
1. **USDA chat-tier fallback integration** — when local DB misses, chat should automatically hit USDA and log the match. Biggest lever for user "it works" moments.
2. **Multi-turn hard-case hardening** — the +15 regression suite is the list of known gaps; systematically close ordinal refs and topic-switch recovery
3. **AI chat analytics (opt-in, local-only)** — capture query → intent → tool outcome locally so we can prioritize real failures over imagined ones, without breaking the privacy-first guarantee
