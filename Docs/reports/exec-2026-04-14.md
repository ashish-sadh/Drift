# Drift Daily Briefing — 2026-04-14

## Headline
Three critical AI chat bugs fixed — food logging now works reliably for voice-style and multi-item inputs.

## Product Snapshot
Drift is an AI-first health tracker — users type what they ate or did, AI handles logging, tracking, and insights. On-device, no cloud. Currently in closed beta on TestFlight.

## What Shipped This Period
- **Users can now say "I want to log my lunch"** — AI correctly understands intent instead of treating it as a food name (P0 fix)
- **Multi-food logging works**: "log 2 eggs and toast and coffee" is parsed and logged as separate items in one message (P0 fix)
- **AI responses are clean** — special model tokens no longer appear in chat output (P0 fix)
- **Voice input no longer loses words** on pauses — committed text is preserved properly
- **Bug reporting made easier** — "Report a Bug" is now the first option in the More tab; anonymous users can submit without signing in

## What's Working Well
- AI chat dual-model pipeline (SmolLM + Gemma) is stable — 19 tools working across food, weight, exercise, supplements, sleep, glucose, and biomarkers
- Momentum on P0 bugs: 3 critical AI chat regressions found and fixed in one cycle
- Autonomous loop is running continuously, tracking tasks via GitHub Issues, and self-enforcing quality gates

## Risks & Blockers
- **AI chat still relies on hardcoded rules** for intent detection — fragile and hard to extend. Design doc #65 proposes switching to LLM-based intent detection (Gemma). Not yet implemented.
- **Food database gap**: 1,544 foods vs. MyFitnessPal's 20M. USDA API integration is the path forward but not yet prioritized.
- **AIRuleEngine has 0% test coverage** — this is core AI logic. Any breakage goes undetected.

## Focus for Next Period
1. **AI Chat reliability (#78–80)** — Input normalization pipeline, 30-query gold set eval, multi-turn context hardening. Directly addresses design doc #65 and prevents P0 regressions.
2. **Design doc #65** — Structural AI chat refactor: replace hardcoded matchers with Gemma-driven intent detection and tool calling. This is the product focus and the highest-leverage architectural improvement.
3. **Coverage hardening (#81–87)** — AIRuleEngine at 0%, WeightTrendService at 52%, NotificationService/BehaviorInsightService need edge case coverage. Quality gate.
