## What's Working Well

## Risks & Blockers
- **2B model attention capacity** — Gemma 4 E2B 2B is at its prompt capacity limit; adding more few-shot examples causes regressions elsewhere; some edge cases (supplement advice vs. status, food sentiment phrases) are at the model's boundary
- **Eval non-determinism** — borderline routing cases flip between runs due to floating-point variability; makes fine-grained prompt tuning difficult
- **State.md food count stale** — Docs show 2,107 foods but actual database is 2,187; documentation drift

## Focus for Next Period
- **Restore eval to 100% pass** — calibrate new test cases to what the 2B model can reliably handle; remove aspirational cases beyond model capacity
- **StaticOverrides for sentiment phrases** — route "feeling bloated", "fasting today" via deterministic pre-LLM layer so LLM doesn't need to handle them
- **Update state.md** — sync food count and capability descriptions to actual shipped state
