# Drift Daily Briefing — 2026-04-22

## Headline
Photo Log overhauled with editable macros, multi-provider model picker, and AI serving units — plus cross-domain correlational queries now live ("do I spike after rice?").

## Product Snapshot
Drift is an AI-first health tracker — users type what they ate or did, AI handles logging, tracking, and insights. On-device, no cloud. Currently in closed beta on TestFlight.

## What Shipped This Period

- **Photo Log overhaul** — Users can now edit macros directly in the review screen, pick AI serving units, see ingredient breakdowns, and choose their preferred AI provider (Anthropic/OpenAI/Gemini) with per-model selection
- **Cross-domain insight tool** — `cross_domain_insight` answers correlational queries like "does my glucose spike after rice?" by analyzing food, glucose, weight, and exercise data together
- **Zero-cal food fix** — Composed foods (e.g. "coffee with milk") were returning 0 kcal; fixed calorie passthrough in the composed-food lookup path
- **FoodLogSheet default amounts** — Default serving amounts now wire correctly to `FoodUnit.defaultAmount` so logging starts with the right quantity
- **Five-bug bundle from user feedback** — Photo log review, stale DB, and key UX issues fixed in a single commit based on direct user reports

## What's Working Well

- **AI pipeline reliability** — 96% overall on per-tool gold set (50 queries); zero regressions introduced in recent AI changes
- **User feedback loop** — Real user bug reports (barcode scanner wrong servings, photo log UX friction) are landing as fixes within the same cycle they're reported
- **Photo Log as differentiator** — Now the only on-device health tracker with a multi-provider AI food photo pipeline (Anthropic/OpenAI/Gemini), editable AI estimates, and ingredient transparency
